# 系统架构概述

## 项目背景
食品安全AI助手是一款移动端应用，帮助消费者通过OCR扫描食品包装获取信息，并由AI大模型分析给出购买建议。

## 技术选型理由

| 组件 | 选型 | 理由 |
|-----|------|------|
| 前端框架 | uni-app | 跨平台(H5/小程序/App)，一套代码多端运行 |
| 前端状态 | Pinia | Vue3官方推荐，轻量级 |
| UI组件 | uview-plus | uni-app生态成熟组件库 |
| 后端框架 | Spring Boot 3.x | 最新的LTS版本，需要JDK 17+ |
| 持久层 | MyBatis-Plus | 国产优秀ORM，减少样板代码 |
| 数据库 | MySQL 8.0 | 支持JSON、窗口函数等新特性 |
| 缓存 | Redis 7.x | 缓存AI响应、减少API调用成本 |
| OCR | 阿里云 | 食品标签识别准确率高 |
| 大模型 | 可配置 | 支持通义千问/文心一言/智谱GLM |
